Description

메인슬라이드배너

$(document).ready() 함수를 사용하여 문서의 DOM이 완전히 로드되면 내부 함수를 실행합니다. 먼저, 슬라이드가 일시정지 상태인지 여부를 저장하는 변수 isPaused를 false로 초기화합니다. 슬라이드 박스(.slidebox)에 마우스를 올리면(mouseenter) isPaused를 true로, 마우스를 떠나면(mouseleave) isPaused를 false로 설정하여 자동 재생을 제어합니다.

이후, slick 플러그인을 사용하여 슬라이드를 초기화합니다. 이를 통해 중앙 정렬 모드(centerMode)를 활성화하고, 중앙 패딩(centerPadding)을 설정하며, 슬라이드 개수(slidesToShow)와 자동 재생(autoplay)을 활성화합니다. 또한, 재생 간격(autoplaySpeed)을 밀리초 단위로 설정하고, 화면 크기에 따른 설정(responsive)을 적용합니다.

슬라이드 박스(.slidebox) 다음에 동적으로 페이저 버튼을 생성합니다. 각 버튼은 data-slide 속성을 가지며, 이를 통해 해당 슬라이드 번호로 이동할 수 있습니다. 버튼을 클릭하면 slickGoTo 메서드를 사용하여 해당 슬라이드로 이동합니다.

슬라이드 변경 전(beforeChange) isPaused가 false일 경우 blurSlides 함수를 호출하여 슬라이드를 블러 처리합니다. 변경 후(afterChange) 중앙 슬라이드의 블러 처리를 제거하고, 다른 슬라이드의 크기를 축소합니다. 또한, 현재 슬라이드에 해당하는 페이저 버튼에 active 클래스를 추가합니다.

이전 버튼을 클릭하면 배경 이미지가 왼쪽 화살표로, 다음 버튼을 클릭하면 오른쪽 화살표로 변경됩니다. 커스텀 이전/다음 버튼을 클릭하면 슬라이드가 이동합니다. 마지막으로 jQuery와 slick 플러그인을 사용하여 슬라이드 쇼를 구현했습니다.

로그인 및 로그아웃

loginBtn과 logoutBtn은 각각 로그인 버튼과 로그아웃 버튼을 선택하는 변수입니다. text는 메인 배너의 h2 요소를, textold는 이 요소의 원래 텍스트를 저장합니다. 초기 설정으로 logoutBtn의 display 속성을 none으로 설정하여 처음에는 보이지 않게 합니다.

초기화 함수인 init는 로컬 스토리지에 사용자 데이터가 없을 경우 기본 사용자 데이터를 설정하는 역할을 합니다. 기본 데이터에는 아이디, 비밀번호, 로그인 상태가 포함되며, 이는 사용자가 처음 페이지를 로드할 때 설정됩니다.

로그인 처리 과정에서는 사용자가 입력한 아이디와 비밀번호를 확인하는데, login 함수는 입력된 정보가 저장된 사용자 정보와 일치할 경우 로그인 성공 메시지를 표시하고 메인 배너의 h2 요소를 사용자 이름과 함께 갱신합니다. 또한, 입력 양식과 로그인 버튼을 숨기고 로그아웃 버튼을 표시하며, 사용자의 isLoggedIn 상태를 true로 설정하고 로컬 스토리지에 업데이트된 사용자 정보를 저장합니다.

로그아웃 처리 과정에서는 logout 함수가 사용자가 로그인 상태일 때 로그아웃을 처리합니다. 로그아웃 성공 메시지를 표시하고 사용자의 isLoggedIn 상태를 false로 변경하여 로컬 스토리지에 저장하며, 메인 배너의 h2 텍스트를 원래 상태로 복원하고 입력 양식과 로그인 버튼을 다시 표시하며 로그아웃 버튼을 숨깁니다. 또한, 입력 필드를 초기화합니다.

로그인과 로그아웃 버튼에 대한 이벤트 리스너는 로그인 버튼을 클릭하면 login 함수를, 로그아웃 버튼을 클릭하면 logout 함수를 호출합니다. 이때 e.preventDefault()를 사용하여 기본 동작(페이지 새로고침)을 방지합니다. 마지막으로, 페이지가 로드될 때 init 함수를 호출하여 로컬 스토리지에 기본 사용자 데이터를 설정하는데, 이를 통해 페이지가 처음 로드될 때 사용자 데이터가 올바르게 초기화게끔 구현했습니다.

Description

햄버거 메뉴, 검색메뉴

햄버거 메뉴

페이지 로드 시 DOMContentLoaded 이벤트가 발생하면 초기 설정이 완료됩니다. 이 시점에서 햄버거 메뉴와 관련된 다양한 요소들이 선택되고 이벤트 리스너가 추가됩니다.

먼저, hamburger 변수는 .hamburger 클래스를 가진 요소를, hamburgerMenu 변수는 .hamburger-menu 클래스를, closeButton 변수는 .close-button 클래스를 가진 요소를 선택합니다.

햄버거 메뉴 열기/닫기를 위해 hamburger 클릭 시, hamburgerMenu에 active 클래스를 토글하고, document.body에 hamburger-menu-open 클래스를 토글합니다.

closeButton 클릭 시에는 이벤트 전파를 방지하고, hamburgerMenu에서 active 클래스를 제거하며, document.body에서 hamburger-menu-open 클래스를 제거합니다.

hamburgerMenu 클릭 시에도 이벤트 전파를 방지합니다. 문서의 다른 부분을 클릭하면 hamburgerMenu에서 active 클래스를 제거하고, document.body에서 hamburger-menu-open 클래스를 제거합니다.

서브 메뉴 토글을 위해 menuItems 변수는 .menu > ul > li 요소들을 선택합니다. 각 메뉴 아이템을 클릭하면 자식 요소인 .sub-menu의 active 클래스를 토글하게끔 구현했습니다.

검색메뉴

페페이지 로드 시 DOMContentLoaded 이벤트가 발생하면 검색 기능과 관련된 요소들이 선택되고 이벤트 리스너가 추가됩니다.

검색 메뉴 열기/닫기를 위해 searchButton 클릭 시, searchMenu에 active 클래스를 토글하고, mainContent에 blur 클래스를 토글합니다. closeButton 클릭 시에는 searchMenu에서 active 클래스를 제거하고, mainContent에서 blur 클래스를 제거하게끔 구현했습니다.

스크롤 애니메이션

스크롤 애니메이션은 window.onscroll을 활용하여, 브라우저 창에서 스크롤 이벤트가 발생할 때마다 특정 함수가 실행되도록 설정하였습니다.

먼저 window.pageYOffset을 이용하여 스크롤된 높이 정보를 height 변수에 저장하였고, window.innerWidth를 사용하여 화면의 너비를 width 변수에 저장했습니다.

각 섹션의 요소들을 대상으로 if-else 조건문을 사용하여 다양한 화면 크기와 스크롤 위치에 따라 스타일을 동적으로 적용합니다.

예를 들어, sec1Title이라는 변수를 대상으로, 화면의 너비가 2200px 이상인 경우와 그렇지 않은 경우를 구분하여 조건적으로 스타일을 적용합니다.

화면 높이가 특정 값 이상인 경우와 그렇지 않은 경우에 따라 opacity와 transform 스타일을 변경합니다. 이러한 조건문을 통해 다양한 화면 크기와 스크롤 위치에서 각 섹션에 다른 스타일을 적용할 수 있습니다.

이러한 방식으로 반응형 디자인에서 설정한 조건에 따라 각 섹션이 스크롤 위치와 화면 너비에 맞춰 정상적으로 작동하도록 구현하였습니다.

프로젝트 스크롤 효과

프로젝트 스크롤 효과를 구현하기 위해 gsap과 ScrollTrigger를 사용하였습니다.

먼저, gsap.utils.toArray('.project_scroll')을 통해 .project_scroll 클래스를 가진 모든 요소들을 배열로 변환하여 projects 변수에 저장합니다.

이후, ScrollTrigger.create를 사용하여 스크롤 트리거를 설정합니다. trigger 속성에 projects 배열을 지정하여, 각 프로젝트 요소가 스크롤 트리거로 작동하도록 설정합니다.

start와 end 속성은 스크롤 위치를 지정하여 애니메이션의 시작과 끝을 정의합니다.

pinSpacing 속성을 false로 설정하여 고정된 요소 주위에 여백이 생기지 않도록 하였고, scrub 속성을 0.5로 설정하여 스크롤 시 애니메이션이 부드럽게 진행되도록 하였습니다.

onEnter 콜백 함수로 pipScroll 함수를 호출하여 요소가 뷰포트에 들어올 때 애니메이션을 시작합니다.

pipScroll 함수 내에서는 jQuery를 사용하여 .mockup 클래스를 가진 모든 요소들을 선택합니다.

device 요소에 대해 마우스 이벤트를 설정하여, mouseenter 이벤트 시 화면의 높이 차이만큼 애니메이션으로 위로 이동시키고, mouseleave 이벤트 시 원래 위치로 돌아가도록 하였습니다

이를 통해 사용자가 프로젝트 요소에 마우스를 올리면 부드럽게 스크롤하는 애니메이션 효과를 구현하였습니다.
